## Metadata & Migration Tools

These scripts assist users migrating from or integrating with **Upload-Assistant** workflows.

### `copy_useful_ua_cache.py`

This script copies useful metadata and cache files generated by Upload-Assistant (UA) from its temporary directory to `upbrr`'s temporary folder.

- **What it copies:**
  - **BDInfo reports:** Files named `Disc*_FULL.txt` which are then parsed and mapped dynamically.
  - **Screenshot JSON uploads:** `image_data.json` (regular screenshots) and `menu_images.json` (disc menu screenshots). `upbrr` reads these JSON files upon release preparation and automatically imports the screenshot URLs, order, sizes, and hosts into the `upbrr` database.
- **Features:**
  - Replaces spaces in directory and file names with underscores (conforming to `upbrr` standards).
  - Skips empty files (0 bytes).
  - Offers a `--dry-run` flag to preview copy operations without writing to disk.
- **Usage:**
  ```bash
  python scripts/copy_useful_ua_cache.py --source "/path/to/Upload-Assistant/tmp" --destination "~/.upbrr/tmp"
  ```
  Use `--dry-run` or `-n` for a preview, and `--force` or `-f` to overwrite existing target files.

### `convert_ua_config.py`

Converts a legacy Upload-Assistant Python configuration (`config.py`) to the YAML configuration structure used by `upbrr`.

- **Usage:**
  ```bash
  python scripts/convert_ua_config.py "/path/to/Upload-Assistant/data/config.py" -o "./config.converted.yaml"
  ```
  Once converted, the configuration can be imported directly into `upbrr` via the CLI or GUI.

## Build & Sync Scripts

These scripts compile the frontend application, sync production assets, and build the final executable binary.

### `build.ps1` / `build.sh`

A full build wrapper that compiles both the frontend and backend in one step.

- **Execution Flow:**
  1. Installs npm packages and builds the production frontend (`gui/frontend`).
  2. Clears the Go GUI embedded assets directory (`internal/guiapp/assets`) and copies the compiled frontend bundle into it.
  3. Triggers the Go compiler to build the CLI binary (`upbrr` / `upbrr.exe`) in the `./dist` folder.
- **Usage (Windows PowerShell):**
  ```powershell
  .\scripts\build.ps1
  ```
- **Usage (Unix/Linux Bash):**
  ```bash
  ./scripts/build.sh
  ```

### `sync-frontend-assets.ps1` / `sync-frontend-assets.sh`

A lightweight script that copies pre-compiled frontend assets from `gui/frontend/dist` directly to `internal/guiapp/assets`.

- **Usage:** Useful when you've compiled the frontend separately or are developing with Wails and want to ensure Go's embedded filesystem assets are synced without running a full build.
